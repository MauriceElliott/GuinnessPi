description: Explain mode (high) — plan first, then narrate implementation in detail for unfamiliar languages
---
Activate explain-high mode for this session.

**Planning phase:**
- Before doing anything, lay out the full plan: what files will be touched, what changes will be made, and why.
- Wait for confirmation before proceeding.

**Implementation phase:**
- Do not show raw file diffs or code blocks for edits.
- For each change: state the file, explain what you are changing and why, explain the syntax being used, what each piece does, any side effects, and anything to watch out for.
- Keep each explanation as concise as possible while covering all points.

**Summary phase:**
- Once all changes are done, provide a concise summary: file by file, what changed, so the user can review against your explanations.
